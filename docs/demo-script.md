# 面试展示脚本

这个脚本用于面试或作品集 walkthrough。

## 1. 启动项目

```bash
uvicorn server.main:app --reload --port 9120
```

打开：

```text
http://127.0.0.1:9120
```

## 2. 先讲问题

Agent 系统很难排查，因为最终回答会隐藏中间链路：LLM 调用、工具调用、Skill 触发、重试、失败和耗时都不直观。

## 3. 讲核心设计

我把 Agent 的每个关键运行步骤转换成结构化事件，并用 `trace_id` 串起来。

事件同时写入：

- JSONL：方便导出和离线分析。
- SQLite：方便本地看板快速查询。

## 4. 演示看板

可以按这个顺序讲：

- 切换时间范围：1 小时、24 小时、7 天、全部。
- 切换样例场景：正常代码修复、权限失败排查、工具超时重试、Skill 触发分析。
- 看 KPI：事件数、trace 数、工具调用、失败次数、LLM 平均耗时。
- 先看高频 Skill 细化优先级，说明频率越高越值得拆细流程、补工具链和产品化。
- 再看部门内专项 Agent 候选，说明多个 Skill 如何组合成一个全流程专项 Agent。
- 打开一条 trace，先讲顶部的用户请求摘要和任务故事，再展示完整时间线。
- 看 Trace 里的部门内专项化判断，说明单次任务如何归入部门通用 Agent 的能力候选池。
- 看失败原因分类和建议动作，说明它如何从“发现失败”推进到“优化 Agent”。
- 看工具性能，说明如何发现慢工具或高错误率工具。
- 点击导出 JSON，浏览器会直接下载当前时间范围内的事件数据。

## 5. 连接到真实项目

这个独立仓库是展示版；真实集成版本在 Hermes Agent fork 分支里。真实版本通过 hook 采集 LLM、Tool、Skill 和任务结果事件。

可以这样总结：

```text
观测解决 Agent 黑盒问题
Trace 复盘单次任务链路
高频 Skill 口径帮助判断哪些能力最值得细化
Skill 组合分析帮助判断哪些能力能拼成全流程专项 Agent
失败分类、建议动作和工具性能帮助定位优化点
本地存储让它可解释、可导出、可继续扩展成测评体系
```
